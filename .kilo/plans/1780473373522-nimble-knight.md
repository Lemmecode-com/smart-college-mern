# Plan: Fix React Duplicate Key Warnings Across Frontend

## Objective
Identify and fix all React duplicate key warnings in the `frontend/src` codebase by replacing unsafe key patterns with stable unique keys.

## Research Summary

### Findings from Codebase Audit

**Total `key={` matches found:** 282 across 136 distinct locations

**Unsafe key patterns identified (9 locations):**

| # | File | Line | Current Key | Risk | Issue |
|---|------|------|-------------|------|-------|
| 1 | `frontend/src/pages/dashboard/Teacher/Timetable/WeeklyTimetable.jsx` | 986 | `key={s._id ? s._id : \`subject-fallback-${index}\`}` | 🟡 Medium | Has fallback but verbose; can be simplified |
| 2 | `frontend/src/pages/dashboard/PlatformSupport/SystemLogs.jsx` | 281 | `key={log._id || idx}` | 🟡 Medium | Falls back to index if `_id` missing |
| 3 | `frontend/src/pages/dashboard/College-Admin/ViewFeeStructure.jsx` | 312 | `key={installment._id || index}` | 🟡 Medium | Falls back to index |
| 4 | `frontend/src/pages/dashboard/College-Admin/ViewFeeStructure.jsx` | 355 | `key={installment._id || index}` | 🟡 Medium | Same as above |
| 5 | `frontend/src/pages/dashboard/College-Admin/PaymentHistory.jsx` | 1397 | `key={record.student_id?._id || \`student-${idx}\`}` | 🟡 Medium | Nested fallback |
| 6 | `frontend/src/pages/dashboard/Teacher/Timetable/MyTimetable.jsx` | 702 | `key={slot._id || slotIndex}` | 🟡 Medium | Falls back to slotIndex |
| 7 | `frontend/src/pages/dashboard/Teacher/Timetable/MySchedule.jsx` | 1521 | `key={slot._id || time}` | 🟡 Medium | Falls back to time string |
| 8 | `frontend/src/pages/dashboard/Teacher/SessionDetails.jsx` | 909 | `key={record._id || idx}` | 🟡 Medium | Falls back to index |
| 9 | `frontend/src/pages/dashboard/College-Admin/Reports/ReportDashboard.jsx` | 1128 | `key={student._id || student.name}` | 🟢 Low | Has name fallback but names may not be unique |

**Skeleton/loading placeholders using `key={i}` (33 instances):**
- These are typically for animated skeleton loaders where the list is static and indices are safe
- **Risk: Low** — these are placeholder UI elements, not data-driven lists
- Can leave as-is or improve for consistency

**Primary suspect for duplicate key warnings:**
The `key={s._id ? s._id : \`subject-fallback-${index}\`}` pattern in `WeeklyTimetable.jsx` and similar fallback patterns are the most likely culprits when `_id` is undefined/null, causing duplicate index-based keys.

## Implementation Plan

### Step 1: Fix WeeklyTimetable.jsx (Issue #6)
**File:** `frontend/src/pages/dashboard/Teacher/Timetable/WeeklyTimetable.jsx`
**Line:** 986
**Before:**
```jsx
<option key={s._id ? s._id : `subject-fallback-${index}`} value={s._id || ""}>
```
**After:**
