# HOD Workflow Unification Plan

## Current State Analysis

### Two Separate HOD Workflows Exist

| Workflow | Endpoint | File | Lines | Creates | Updates |
|----------|----------|------|-------|---------|---------|
| **Create Staff (HOD)** | `POST /college/staff` | `staff.controller.js` | 163-219 | User(ROLE.HOD), Teacher, Department.hod_id | None |
| **Assign HOD** | `PUT /departments/:id/assign-hod` | `department.controller.js` | 163-219 | None | Department.hod_id, User.role |

### Current Assign HOD Flow

```
College Admin opens department → Clicks "Assign HOD" → AssignHod.jsx
  ├─ Fetches /teachers (all teachers)
  ├─ Filters to teachers in this department (line 49-51)
  └─ POST /departments/:id/assign-hod with { teacher_id }
      ├─ Validates teacher belongs to department (line 181)
      ├─ Sets department.hod_id = teacher._id (line 194)
      └─ Sets user.role = "HOD" (line 199)
```

### Current Staff Creation (HOD) Flow

```
College Admin opens /college/staff/create → CreateStaff.jsx
  ├─ Selects role="HOD"
  ├─ Selects departmentId
  └─ POST /college/staff
      ├─ Creates User with role="HOD"
      ├─ Creates StaffProfile
      └─ Creates Teacher record (line 173-200)
          └─ Sets department.hod_id = teacher._id (line 203-207)
```

## Problems Identified

### 1. User.role Rollback Issue
- **When reassigning HOD**: Old HOD's `user.role` remains "HOD" (never reset to "TEACHER")
- **Location**: `department.controller.js` line 177-186 - captures `previousHodId` but never uses it to rollback

### 2. Multiple HOD Risk
- **No validation prevents**: One teacher being HOD of multiple departments
- **No validation prevents**: Department already having an HOD (overwrites silently)

### 3. Duplicate HOD Blocking (Wrong Direction)
- `staff.controller.js` line 92-101: Prevents creating HOD if teacher email already exists
- **Should be opposite**: Prevent HOD creation if department already has HOD

### 4. Missing Teacher Record Check
- `department.controller.js` line 178: Doesn't check if selected teacher already has ROLE.HOD

## New Unified Workflow

```
College Admin creates staff → Selects HOD role → Selects department
  ├─ POST /college/staff
  │   ├─ Creates User(role="HOD")
  │   ├─ Creates StaffProfile
  │   ├─ Creates Teacher
  │   └─ Sets Department.hod_id
  │
  └─ BEFORE creation:
      ├─ Check if department already has HOD → return error
      ├─ Check if teacher exists with HOD role → prevent duplicate
      └─ If valid → proceed with creation
```

### Validation Rules

```
CREATE STAFF (role=HOD):
  1. departmentId must exist and be ACTIVE
  2. Check: Department.hod_id === null → if not, return "Department already has HOD"
  3. Check: No existing Teacher with role="HOD" in same department
  4. Create User(role="HOD")
  5. Create Teacher record
  6. Create StaffProfile
  7. Set Department.hod_id = new Teacher._id

ASSIGN HOD TO DEPARTMENT (keep for backward compatibility):
  1. Check: Teacher exists and belongs to same department
  2. Check: Department has existing HOD?
     - If yes: Get old HOD's User record → set role="TEACHER", clear Department.hod_id
  3. Set Department.hod_id = teacher._id
  4. Set User.role = "HOD"
```

# Remove HOD / HOD Demotion Feature Plan

## Requirements Analysis

- Teacher record MUST remain intact
- Subject assignments MUST remain
- Timetable assignments MUST remain
- Attendance history MUST remain
- User.role MUST revert from HOD → TEACHER
- Department can temporarily exist without HOD
- No data deletion

## Backend Changes

### New Function: `removeHOD()`

| File | Function | Purpose |
|------|----------|---------|
| `backend/src/controllers/department.controller.js` | `removeHOD` | New function to remove HOD from department |

### API Endpoint

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| DELETE | `/departments/:id/hod` | COLLEGE_ADMIN | Removes HOD from department, reverts to TEACHER |

### Validation Rules

1. Department must exist
2. Department must have an HOD assigned (`hod_id !== null`)
3. On success:
   - Set `department.hod_id = null`
   - Find Teacher via `hod_id`
   - Find User via Teacher's `user_id`
   - Set `user.role = "TEACHER"`

### Security Impact

| Risk | Mitigation |
|------|------------|
| HOD loses access immediately | Audit log records the action |
| Cannot demote self | COLLEGE_ADMIN cannot have HOD role |
| Race condition on concurrent demotions | MongoDB document locking on `department.save()` |

### Regression Risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| Department with no HOD breaks middleware | Medium | Add null check in `hod.middleware.js` line 77 |
| HOD still accesses via cached JWT | Low | JWT expires in 24h, role checked on each request |
| Frontend shows "No HOD" state | Low | Update UI to handle missing HOD gracefully |
