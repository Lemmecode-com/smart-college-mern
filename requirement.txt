============== Runtime Requirements ==============
Node.js: >=20.19.0
Recommended Node.js: 20.19.x LTS or 22.12.x+
npm: 10+
Package manager: npm
Important: Use package-lock.json as the source of truth for deployments.

Why Node.js >=20.19.0 is required:
- Frontend Vite 7 requires Node.js ^20.19.0 || >=22.12.0.
- Backend Mongoose 9 requires Node.js >=20.19.0.
- Backend Express 5 requires Node.js >=18, but Mongoose 9 is the stricter requirement.

MongoDB:
- MongoDB Atlas is recommended.
- Use a MongoDB server version compatible with Mongoose 9 / MongoDB Node.js driver 7.

============== Deployment Command Standard ==============
Do not use npm install for repeatable deployments when package-lock.json exists.
Use npm ci so the locked dependency tree is installed exactly.

Backend production install:
cd backend
npm ci --omit=dev
npm start

Frontend production build:
cd frontend
npm ci
npm run build

Frontend production output:
frontend/dist

Frontend hosting:
- Deploy frontend/dist to a static hosting provider, CDN, or web server.
- Vite dev server is only for development, not production.
- Set frontend environment variables before running npm run build because Vite embeds them into the build.

============== Deployment Notes ==============
- package-lock.json must be committed for reproducible deployments.
- Use npm ci instead of npm install in deployment pipelines.
- Frontend environment variables are baked into the build output.
- Backend .env values must exist on the production server/container.
- Do not expose backend secrets in frontend environment variables.
- Use HTTPS for production frontend and backend domains.
- Configure CORS_ORIGINS with the production frontend domain before deployment.
- Ensure MongoDB Atlas network access allows the backend deployment IP.
- Backend production entrypoint is backend/server.js through npm start.
